java正则表达式详解(2)

发表于 2018-03-01 | 分类于 java

4.匹配标记

(1)多行模式(multiline )

表示方式：(?m)(pattern)
当使用^时，^可以匹配\n \r之后的内容，$可以匹配\n \r之前的内容。如：正则表达式”(?m)^(ab)” 可以匹配字符串 ”ab\nab”中的两个ab。通过调用pattern.compile("^(ab)" , Pattern.MULTILINE)与此等效。

(2).单行模式(single line)

表示方式：(?s)(pattern)
默认情况下就是单行模式。

(3).忽略英文字母的大小写

表示方式：(?i)(pattern)
忽视表达式内任意个英文字符的大小写。通过调用pattern.compile(String regex, pattern.case-insensitive )有同效。

(4).忽视注释和空白

表示方式：(?x)(pattern)
忽视表达式内任意个空白和以#打头的注释内容。通过调用pattern.compile(String regex, pattern.COMMENTS )有同效。

5.匹配模式

(1).Greediness（贪婪型）： 最大匹配

X?、X*、X+、X{n，}都是最大匹配。正则的默认情况下就是最大匹配，它们会匹配符合要求的最长的子串。如“<.+>”去匹配“a<tr>aava </tr>abb”只能得到<tr>aava </tr>，而不能得到和。
匹配过程：先匹配到’<’ ,然后 ”.+” 匹配了 ”tr>aava </tr>ab”，发现’>’不能得到匹配。于是从最后返回，找到最近的’>’。

(2)、Reluctant(Laziness)（勉强型）：最小匹配

X??、X*?、X+?、X{n，}?都是最小匹配
它会匹配最短的符合要求的子串。
如“<.+?>”去匹配“a<tr>aava </tr>abb”只能得到<tr>和</tr>，而不能得到<tr>aava </tr>。
匹配过程：’<’匹配之后，’.+’ 每多匹配一个字符，就让 ’>’ 尝试匹配。若’>’得到匹配，立即返回子串。
注意：这个所谓的最短有时候并不是真的最短，如.+?>去匹配上面的串时，首先匹配到的是a<tr>,而不是r>。所以这种匹配最短的方式适用于有明确的左右边界情况，如“<”和“>”。

(3)、Possessive（占有型）：完全匹配

X?+、X*+、X++、X{n，}+等，被称为完全匹配
它和最大匹配一样，一直匹配所有的字符，直到文本的最后，但它不由原路返回。如“<.++>”去匹配“a<tr>aava </tr>abb”什么也得不到。

6.POSIX字符类

Java 支持POSIX字符（仅 US-ASCII）

  • \p{Lower} 小写字母字符：[a-z]
  • \p{Upper} 大写字母字符：[A-Z]
  • \p{ASCII} 所有 ASCII：[\x00-\x7F]
  • \p{Alpha} 字母字符：[\p{Lower}\p{Upper}]
  • \p{Digit} 十进制数字：[0-9]
  • \p{Alnum} 字母数字字符：[\p{Alpha}\p{Digit}]
  • \p{Punct} 标点符号：!”#$%&’()*+,-./:;<=>?@[]^_`{|}~
  • \p{Graph} 可见字符：[\p{Alnum}\p{Punct}]
  • \p{Print} 可打印字符：[\p{Graph}\x20]
  • \p{Blank} 空格或制表符：[ \t]
  • \p{Cntrl} 控制字符：[\x00-\x1F\x7F]
  • \p{XDigit}十六进制数字：[0-9a-fA-F]
  • \p{Space} 空白字符：[ \t\n\x0B\f\r]

7.java中正则相关的类

1.Pattern

1.静态获取对象：static Pattern compile(String regex)
2.主要方法：Matcher matcher(CharSequence input)

2.Matcher

1.匹配方法：
public boolean matches() 从input的开始位置进行匹配，正则只有在匹配上整个字符串时返回true，otherwise false。
public boolean lookingAt() 从input的开始位置进行匹配，匹配任意长度的子串
public boolean find() 从input的任何位置开始匹配，匹配任意长度的子串
2.获取方法：
String group(int group) 根据捕获组数
String group(String name) 根据组名
3．其他：
String replaceAll(String replacement) 字符串的replaceAll方法是对这个方法的调用。

8.js中正则

1.构造方式：

/pattern/[flags] 或new RegExp("pattern",["flags"])
Flags: g （全文查找所有的） i （忽略大小写） m （多行查找）使用多个时，直接连在一起/img

2. 主要方法:

rgExp.compile(pattern, [flags]) 编译正则，重复使用时执行效率更高。
rgExp.exec(str) 执行匹配，返回结果是数组类型
rgExp.test(str) 判断是否匹配成功

3.其他：

stringObj.replace(rgExp, replaceText)
